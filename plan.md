# 本地吃喝玩乐推荐系统 - 总体思路（含技术栈）

## 1. 项目目标
为用户提供一个发现本地商家（餐厅、咖啡厅等）、查看用户点评、购买团购优惠券的 Web 应用。类似大众点评的极简版。

## 2. 技术栈（仅选型，无具体设计）
- **前端**：React 18 + 组件库（Ant Design 或 Bootstrap）+ Axios + React Router
- **后端**：Python 3.10+，FastAPI 框架，SQLAlchemy ORM，PyMySQL 驱动
- **认证**：JWT（JSON Web Token）
- **数据库**：MySQL 8.0
- **环境管理**：Python venv，Node.js npm，`.env` 文件存储敏感信息
- **开发辅助**：Faker 库生成测试数据

## 3. 用户角色
- 普通用户（未登录）：可以浏览商家和团购，但无法发表点评或购买。
- 注册用户（已登录）：可以发表点评、购买团购券、查看自己的订单。
- （无管理员角色，后续可扩展）

## 4. 核心功能模块

### 4.1 用户账户模块
- 注册：设置用户名、密码
- 登录：使用用户名密码登录，系统返回一个凭证（JWT token，后续请求需携带）
- 登录后可访问个人中心（查看自己的订单）

### 4.2 商家浏览模块
- 商家列表页：展示商家的名称、图片、评分、人均消费、类别。支持分页加载。
- 筛选功能：按类别（火锅、日料、咖啡等）、按城市（或区域）筛选商家。
- 商家详情页：展示详细信息（地址、电话、营业时间描述），以及该商家下的所有用户点评、所有团购商品。

### 4.3 点评模块
- 已登录用户可以在某个商家下发表文字点评和 1-5 星评分。
- 商家详情页展示所有用户的点评列表（按时间倒序）。
- 商家的平均评分根据所有用户的评分实时计算并展示。

### 4.4 团购模块
- 每个商家可以发布多个团购商品（标题、原价、团购价、已售数量）。
- 商家详情页展示该商家的团购商品列表。
- 已登录用户可以购买某个团购商品（生成订单，状态为“待使用”）。
- 用户可以在“我的订单”页面查看自己购买的所有团购订单。

### 4.5 订单模块
- 订单包含：购买的团购商品信息、购买时间、状态（待使用/已使用）。简化为“待使用”即可，暂不实现核销功能。

## 5. 业务逻辑要点
- 只有登录用户才能发表点评和购买团购券。
- 一个用户对同一家商家只能发表一次点评（业务规则，可后续增加校验）。
- 商家列表按评分从高到低排序（或者按综合排序，简单实现即可）。
- 团购商品的销量（已售数量）在用户购买后自动+1。

## 6. 非功能需求（供参考，不必全部实现）
- 界面美观、响应式（适配手机和电脑）。
- 后端 API 符合 RESTful 风格。
- 数据持久化使用关系型数据库。
